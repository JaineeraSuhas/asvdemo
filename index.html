Signal Pipeline

How ASV Works

10-stage pipeline from raw muscle signal to spoken word — happening in under 500 milliseconds.

01
EMG Signal Acquisition
Ag/AgCl surface electrodes on masseter (jaw sides) and suprahyoid (under chin) muscles detect microvolt-level electrical activity during silent speech. 3 electrode points. Sampling rate 1000–2000 Hz.
Ag/AgCl Electrodes 1000–2000 Hz sampling 0.1–10 mV range
02
Amplification
INA128 instrumentation amplifier boosts the tiny signal by 1000× while rejecting electromagnetic noise using differential electrode setup.
INA128 Gain = 1000× CMRR > 90 dB
03
Bandpass Filtering (20–500 Hz)
4th-order Butterworth filter removes movement artifacts below 20 Hz and high-frequency noise above 500 Hz. Implemented in both hardware and software.
Butterworth 4th Order 20–500 Hz passband SciPy
04
Notch Filter (50 Hz)
Eliminates Indian power-line interference at 50 Hz using a sharp IIR notch filter. Q factor = 30 for precise rejection without affecting EMG signal.
IIR Notch Q=30 50 Hz rejection
05
Rectification & Envelope Detection
Full-wave rectification followed by 20ms moving-average envelope creates a smooth activation profile showing when muscles are firing.
Full-wave Rectifier 20ms window NumPy
06
Windowing & Segmentation
200ms sliding windows with 50ms overlap. Hanning window applied per segment to reduce spectral leakage before frequency analysis.
200ms window 50ms overlap Hanning window
07
Feature Extraction (~40D vector)
Time-domain: MAV, RMS, ZCR, Waveform Length. Frequency: Mean Power, Median Frequency, STFT. Cepstral: 13 MFCCs. Combined ~40-dimensional feature vector per window.
MAV · RMS · ZCR · WL 13 MFCCs STFT
08
Normalisation & PCA
Z-score normalisation across sessions. Optional PCA reduces to top 20 components for lightweight inference on ESP32 microcontroller.
Z-score norm PCA → 20D Scikit-learn
09
ML Classification
Trained CNN-LSTM model classifies intended word from feature vector. Confidence threshold 0.75 — below that, returns "unclear" to avoid false positives.
CNN-LSTM Hybrid TensorFlow Lite Conf. threshold 0.75
10
Output — Text & Speech
Classified word sent via BLE to phone/PC. TTS engine speaks it aloud. Consecutive duplicate detection (300ms hold-off) prevents stuttered output. Sentence assembled word-by-word.
BLE GATT TTS Engine 300ms debounce Sentence assembly

[ 01 ]
Masseter (Jaw Sides)
Two electrodes placed on the sides of the jaw, just in front of the ears. Feel it flex when you clench your teeth — that's the masseter. Strongest EMG source for jaw movements.
Position: Cheek-jaw junction · Bilateral · 32mm Ag/AgCl
[ 02 ]
Suprahyoid (Under Chin)
One electrode placed directly under the chin on the soft fleshy area. Activates during tongue movement, swallowing, and vowel formation. Key for word differentiation.
Position: Submental region · Midline · Captures tongue muscle signals
[ 03 ]
Throat Reference (Ground)
One neutral ground electrode on the front of the throat, below the Adam's apple. Used as an electrical reference point to cancel common-mode noise from both active channels.
Position: Anterior neck · Reference/GND · Below thyroid cartilage

New Feature

Multilingual EMG

Each language shapes muscles differently. ASV learns every tongue — including phonemes no prior dataset has ever captured.

GL
Multilingual · Cross-Lingual Transfer
One device, every language — starting with India's most spoken
Hindi retroflex consonants (ट, ड, ण) produce a distinct suprahyoid burst from tongue-tip curling against the palate. Kannada's ಳ (retroflex lateral) has never existed in any EMG dataset — making ASV's corpus a potential world-first. Transfer learning means the Kannada model trains in hours, not months, by inheriting English and Hindi feature extractors.
EN English (Base) HI Hindi KN Kannada — World First ✦ TA Tamil (Phase 5) JP Japanese (Phase 5)
Dataset Scale
20 speakers
50 core words × 30 reps each = 1,500 recordings per speaker, 30,000 labelled segments per language.
World-First Phoneme
Kannada ಳ
The retroflex lateral — zero prior EMG representation globally. New output class expanding softmax from N → N+1.
Transfer Learning
<20 epochs
Freeze CNN layers (English features), unfreeze last 3 LSTM + dense. Fine-tune at 1/10× LR on target language data.
Hindi Key Insight
त vs ट pairs
Record dental (त) and retroflex (ट) versions of same consonant to train the masseter retraction difference in the model.
Switch Latency
<2 seconds
On-device language switching — multiple TFLite models cached in ESP32 flash. No cloud round-trip required.
Model Size (each)
<800 KB
INT8 quantised via TFLite Converter. All language models fit simultaneously in ESP32-S3's 8MB PSRAM partition.
Training Pipeline

8-Step Model Training — Start Today

Click any step to expand the full technical detail.

01 Data Collection Script pyserial · ESP32 · CSV ▼
Write a Python script using pyserial to pull raw ADC values from the ESP32 over USB. Display a word on screen, wait 1 second, then record 500 ms of EMG at 1 kHz. Auto-label and save as speaker01_hindi_roti_rep03.csv. Format: [speaker_id, word, rep, ch1[], ch2[], ch3[]]. Run for all 20 speakers, all words, 30 reps each.
python collect.py --speaker 01 --lang hindi --words vocab_hindi.txt --reps 30 --port COM3
02 Preprocessing — 10-Stage Pipeline scipy · numpy · 40D vector ▼
Apply the standard pipeline: bandpass 20–450 Hz → 60 Hz notch → RMS windowing. Extract the 40-dimensional feature vector per window: 13 MFCCs, 6 RMS bands, zero crossing rate, 4 frequency peaks, waveform length, slope sign changes, Wilson amplitude. These 40 numbers are the input to every model in the pipeline.
features = extract_40d(segment) # [mfcc×13, rms×6, zcr, freq×4, wl, ssc, wa]
03 English Base Model — CNN-LSTM TensorFlow · INT8 · <800KB ▼
Train a CNN-LSTM on your 40D feature vectors. The CNN extracts spatial patterns across channels; the LSTM captures temporal dependencies across the 200 ms window. Target 93% on held-out test set. Quantise to INT8 using TFLite Converter. Final model must be under 800 KB to fit ESP32 PSRAM.
converter = tf.lite.TFLiteConverter.from_saved_model(model_path) → converter.optimizations = [tf.lite.Optimize.DEFAULT]
04 Hindi Transfer Learning Freeze CNN → Unfreeze LSTM+Dense ▼
Load English model weights. Freeze all CNN layers (they've learned generic EMG feature extraction). Unfreeze only the final 3 LSTM + dense layers. Fine-tune on 30,000 Hindi subvocal segments for 20 epochs at 1/10th the original learning rate. The model learns Hindi-specific phoneme patterns without forgetting English feature extraction.
model.layers[:cnn_depth].trainable = False → model.compile(lr=1e-4) → model.fit(hindi_ds, epochs=20)
05 Kannada Transfer + ಳ New Class World-First · Softmax N→N+1 ▼
Start from the Hindi model (shares many retroflex structures). Add ಳ as a new output class by expanding the final softmax layer from N to N+1 classes. Re-initialise only the new neuron weights. Fine-tune the last 5 layers on Kannada data. The ಳ class alone constitutes a unique patent-eligible dataset claim — the first labelled subvocal EMG for this phoneme.
model.layers[-1] = Dense(N+1, activation='softmax') → fine_tune(kannada_ds, last_n_layers=5)
06 SOS Detector — Parallel Binary Classifier 12KB model · ESP32 interrupt thread ▼
Train a separate, parallel lightweight binary classifier — not the word recogniser. Input: masseter RMS energy time-series. Positive samples: hard jaw clenches sustained ≥1.5 s. Negative samples: normal speech, chewing, talking, yawning. This model is only 12 KB and runs as an interrupt thread on ESP32 at all times, completely independent of the word recognition pipeline.
sos_model = train_binary(positives=jaw_clenches, negatives=normal_speech) → quantise → 12KB .tflite
07 Dive Vocabulary (20 words, 97%+ target) English base → tactical dive terms ▼
Fine-tune the English base model on 20 tactical dive commands recorded from divers in both wet and dry sessions. Short, distinct words — "stop", "up", "abort", "surface", "buddy", "air", "okay", "danger", "hold", "go" — have very different EMG signatures. Small vocabulary means the model converges fast. Target: ≥97% accuracy with only 20 output classes.
dive_words = ["stop","up","abort","surface","buddy","air","okay","danger","hold","go",...] # 20 classes
08 Army Vocabulary — Stress-Condition Recording Post-exertion EMG · adrenaline compensation ▼
Record tactical words after 50 push-ups to simulate adrenaline-state jaw tension changes. A model trained at rest fails in the field — masseter and suprahyoid activation levels shift under physical stress. Collect both rest-state and stress-state samples; train with augmentation mixing both. The stress-aware dataset is critical for military reliability and constitutes a novel training methodology.
army_ds = concat(rest_recordings, post_exercise_recordings) → train(augment=True, stress_weight=0.6)

Emergency System

SOS Jaw-Clench Emergency

No new hardware. No extra electrodes. Just a parallel firmware thread that watches your masseter 24/7 — and sends help when you can't speak.

⊘
EMERGENCY SYSTEM — ALWAYS ACTIVE
3× jaw clench → emergency alert → GPS broadcast
The SOS system requires zero new hardware — it is a pure firmware and app update. The masseter electrode is already present. A parallel detection thread on the ESP32 watches for a 3× RMS spike sustained for 1.5 seconds. The detection is completely independent of the word recognition pipeline — it runs as a lightweight interrupt thread using a 12 KB binary classifier. Once triggered, the device vibrates 3× to alert the user, then waits 10 seconds for a phone tap to cancel before broadcasting the emergency payload with GPS coordinates to registered caretakers and nearby medical facilities.

Visual representation only — the jaw-clench trigger is a single sustained bilateral masseter contraction, not morse keying.
Trigger
3× RMS Spike
Bilateral masseter clench ≥1.5 s sustained. Detected by the parallel SOS thread, not the word recogniser.
Cancel Window
10 seconds
Device vibrates 3× immediately on trigger. User has 10 s to tap phone to cancel — prevents accidental alerts.
Model Size
12 KB
Binary classifier (SOS / not-SOS) on masseter RMS time-series. Runs as ISR thread, always on, zero impact on word recognition.
Payload
GPS + SMS
Emergency alert with live GPS coordinates broadcast to registered caretakers + nearest hospital via phone cellular.
Training Data
Pos / Neg split
Positive: hard jaw clenches ≥1.5 s. Negative: normal speech, chewing, yawning, talking. Balanced dataset, threshold=0.92.
